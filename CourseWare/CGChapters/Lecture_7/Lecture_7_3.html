计算机图形学

第七章(3) 填充算法

多边形填充

问题： 给定多边形的顶点序列，如何“点亮”多边形内部的所有像素？

这涉及到两个核心问题：

  1. 判断点在多边形内： 哪些像素属于多边形内部？
  2. 填充策略： 如何高效地遍历并着色这些像素？

内外测试 (Inside/Outside Tests)

对于简单凸多边形，很容易判断。但对于凹多边形或自相交多边形，需要通用规则。

奇偶规则 (Odd-Even Rule)

方法： 从任意一点 $P$ 向无穷远处发出一条射线。

统计射线与多边形边界的交点个数。

  • 奇数 (Odd)： 点 $P$ 在多边形内部。
  • 偶数 (Even)： 点 $P$ 在多边形外部。

注意：需要处理射线经过顶点或与边重合的特殊情况。

非零环绕数规则 (Non-Zero Winding Number Rule)

方法： 同样发射射线，但考虑边界的方向。

  • 初始化环绕数 $W = 0$。
  • 当边界从右向左穿过射线时，$W = W + 1$。
  • 当边界从左向右穿过射线时，$W = W - 1$。

结论： 若 $W \neq 0$，则点在内部；若 $W = 0$，则点在外部。

扫描线多边形填充算法

核心思想： 利用扫描线（水平线）与多边形边界的交点具有成对性（Parity）。

对于每一条扫描线 $y$：

  1. 求出扫描线与所有多边形边的交点。
  2. 将交点按 $x$ 坐标递增排序。
  3. 将交点两两配对 $(x_1, x_2), (x_3, x_4), \dots$。
  4. 填充每对交点之间的区间 $[x_{2k-1}, x_{2k}]$。

数据结构

为了避免每行都重新计算所有边，引入两个关键数据结构：

边表 (Edge Table, ET)

按边的下端点 $y_{min}$ 分类存储所有边。

每条边记录：$y_{max}, x_{ymin}, \frac{1}{m}$ (斜率倒数)。

活化边表 (Active Edge Table, AET)

存储当前扫描线与之相交的边。

随着扫描线移动，边从 ET 进入 AET，当扫描线超过 $y_{max}$ 时从 AET 移除。

算法流程

  1. 初始化 ET，置 AET 为空。
  2. 从 $y = y_{min}$ 开始向上扫描：
    • 将 ET 中 $y_{min} = y$ 的边加入 AET。
    • 移除 AET 中 $y_{max} = y$ 的边。
    • 对 AET 中的边按交点 $x$ 排序。
    • 两两配对填充。
    • 更新 AET 中每条边的 $x$ 值：$x_{new} = x_{old} + \frac{1}{m}$。

区域填充 (种子填充)

适用场景： 区域由边界颜色或特定内部颜色定义，而非顶点序列（如画图软件中的油漆桶工具）。

输入： 种子点 $(x, y)$，目标颜色。

泛滥填充 (Flood Fill)

定义： 将区域内所有旧颜色的像素替换为新颜色。


void FloodFill(int x, int y, int oldColor, int newColor) {
    if (GetPixel(x, y) == oldColor) {
        SetPixel(x, y, newColor);
        FloodFill(x + 1, y, oldColor, newColor);
        FloodFill(x - 1, y, oldColor, newColor);
        FloodFill(x, y + 1, oldColor, newColor);
        FloodFill(x, y - 1, oldColor, newColor);
    }
}

通常使用栈来实现非递归版本，以避免栈溢出。

连通性 (Connectivity)

4-连通

只考虑上下左右四个邻居。

无法穿过对角线方向的缝隙。

8-连通

考虑周围8个邻居（含对角线）。

可以填充更复杂的区域，但可能“泄露”到外部（如果边界是对角线连接的）。

总结

  • 奇偶规则和非零环绕数规则用于判断点在多边形内外。
  • 扫描线算法利用边的连贯性高效填充多边形，是矢量图形渲染的基础。
  • 种子填充算法（如泛滥填充）基于像素连通性，常用于交互式绘图应用。
